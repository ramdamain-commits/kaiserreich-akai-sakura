# Repo Rules

- 親 `C:\Users\ramda\projects\CLAUDE.md` を先に適用する
- 正本は `manuscript/` 配下の各章 `.md` ファイル。設定資料は `kaiserreich-*-setting.md` と `kaiserreich-*-reference.md`
- 3稿の改稿計画は `revision-plan-3rd.md` を参照する

## 設計制約

- **R2制約**: 朴の内面描写は最小限。感情は行動・身体・沈黙で描く
- **「振り返らなかった」**: 3回限定（ゼロ章§1変形、ゼロ章§5、第2章§5）
- **鬼頭6シーン**: 第2章（起源）、第3章、第4章、第5章、第6章、終章
- **紙モチーフ**: 帆布小袋→方眼紙→連邦憲法→看板
- **桜モチーフ**: ゼロ章§5（種）→ 終章§4（回収）
- **1戦闘1損耗**: 各戦闘に死者数・装備損失・時刻のいずれか1数字

## 編集ルール

- 原稿の編集は Claude Code がメイン担当。Codex は読み取り専用レビュー（`AGENTS.md` 参照）
- 基幹数値（兵力・死者数・年齢・日付）を変更したら `manuscript/` 全ファイルを grep して矛盾がないか確認する
- 章の追加・削除・統合は `revision-plan-3rd.md` を先に更新してから原稿に反映する

## 執筆プロセス

- 字数に引っ張られて水増ししない。加筆は意味のある内容のみ
- 新章を設計するときは「機能（何を担うか）＋具体シーン2つ」を先に決める
- Phase 開始時は既存構成を見直してから着手する（前 Phase の残課題を先に拾う）
- レビューは 3〜5 章単位でバッチ実行。レビュワーは3名（構造設計・R2監査・ポストコロニアル）で十分
- 査読フロー: 第三者レビュー → チーム議論 → 反映。追記は1項目あたり最大3行
- Markdown テーブル書き込み後はレンダリング確認（blockquote が行を分断しないか注意）
- 横断整合性チェックは ctx_batch_execute で複数ファイルを一括検索する
